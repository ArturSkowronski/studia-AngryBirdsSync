Angry Birds Sync

Angry Birds Sync API documentation.

Aplikacja mobilna pozwala na synchronizacje w chmurze wyników rozgrywki gry Angry Birds.

Celem zadania było stworzenie infrastruktury sieciowej przeznaczonej do umożliwienia synchronizowania wyników pomiędzy urządzeniami. Oprogramowanie jest może służyć jako część większego projektu, w obecnym stadium wszelkie wyniki, które w docelowym projekcie byłyby pobierane z gry, są generowane pseudolosowo na bazie predefiniowanych struktur danych.

Android

Użyte biblioteki zewnętrzne:

  • AppCompat
  • Retrofit
  • Apache Commons
  • HTTPOk

Wymagania Dodatkowe

Wspieranie przez telefon Google APIs.

Sposób działania

  • Aplikacja Androidowa loguje użytkownika
  • Aplikacja podpisuje requesty kluczem publicznym użytkownika
  • Aplikacja umożliwia generowanie nowego pakietu danych
  • Aplikacja umożliwia wysłanie nowego zestawu danych w celu zaktualizowanie stanu chmury
  • Aplikacja umożliwia przeglądnięcie historii wyników

Dodatkowe informacje

Aplikacja rozpoznaje urządzenie użytkownika za pomocą numeru IMEI oraz konta użytkownika

WebServer

Wersja App Engine SDK: 1.9.6 Wersja Datastore API: 2.3.5

Użyte biblioteki zewnętrzne:

  • ApacheCommons
  • Objectify
  • DataNucleus

Kroki Implementacyjne

  • Utworzenie nowego projektu Google App Engine
  • Ustawienie parametrów skalowania - w naszym przypadku najtańsze rozwiązanie z replikacją opóźnioną o 60 sekund między instancjami, 3 instancje
  • Aktywowanie Android API w ramach projektu
  • Wygenerowanie kluczy aplikacji po stronie klienta/serwera
  • Utworzenie Modelu i wygenerowanie kluczy do bazy danych
  • Utworzenie projektu
  • Deployment projektu za pomocą wcześniej utworzonych kluczy
